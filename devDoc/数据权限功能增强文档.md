# 数据权限功能增强文档

## 📋 概述

本文档记录了数据权限功能的增强和优化工作。本次增强在不破坏现有功能的前提下，提升了数据权限控制的性能、灵活性和易用性。

## 🎯 增强目标

1. **性能优化** - 提升部门树查询和数据权限计算的效率
2. **功能完善** - 支持5种数据权限等级，提供完整的工具方法
3. **易用性提升** - 自动查询条件构建，简化开发工作
4. **兼容性保证** - 不破坏现有接口，支持平滑升级

## 🏗️ 架构设计

### 数据权限等级（5种）

| 等级 | 代码                   | 说明                     | 适用场景   |
| ---- | ---------------------- | ------------------------ | ---------- |
| 1    | `全部数据权限`         | 可查看所有数据           | 超级管理员 |
| 2    | `自定义数据权限`       | 可查看指定部门的数据     | 部门管理员 |
| 3    | `本部门数据权限`       | 仅可查看本部门数据       | 部门主管   |
| 4    | `本部门及以下数据权限` | 可查看本部门及子部门数据 | 区域经理   |
| 5    | `仅本人数据权限`       | 仅可查看自己的数据       | 普通员工   |

### 核心组件

1. **部门模型增强** (`src/models/system/dept/dept.ts`)
   - 添加 `level` 字段记录部门层级
   - 添加 `ancestors` 字段记录部门路径
   - 实现保存中间件自动计算层级和路径

2. **部门服务工具类** (`src/services/system/dept.service.ts`)
   - 单例模式设计
   - 提供高效的部门树查询方法
   - 支持批量获取子部门ID

3. **用户数据权限计算服务** (`src/services/system/userRole.service.ts`)
   - 根据用户角色计算数据权限
   - 自动构建查询过滤器
   - 支持权限检查和验证

4. **数据权限中间件** (`src/middlewares/dataScope.ts`)
   - 完全重写，性能优化
   - 自动构建MongoDB查询条件
   - 提供工具方法：`getQueryCondition`, `hasDeptAccess`, `hasUserAccess`

### 2. 高性能部门查询算法

```typescript
// 使用正则匹配优化子部门查询
async getChildrenDepts(deptId: string): Promise<string[]> {
  // 构建正则表达式：查找 ancestors 中包含目标部门ID的部门
  const regexPattern = new RegExp(`,${deptId}(,|$)`);

  const childDepts = await DeptModel.find({
    ancestors: { $regex: regexPattern },
    delFlag: '0',
    status: '0',
  }).select('_id').lean();

  return [deptId, ...childDepts.map(dept => dept._id.toString())];
}
```

### 3. 自动查询条件构建

```typescript
// 数据权限中间件自动构建查询条件
const getQueryCondition = function (modelName?: string) {
  // 根据权限等级自动构建过滤器
  switch (this.dataScope) {
    case '1': // 全部数据权限
      return {};
    case '2': // 自定义数据权限
      return { deptId: { $in: this.deptIds } };
    case '3': // 本部门数据权限
      return { deptId: this.userDeptId };
    case '4': // 本部门及以下数据权限
      const childDeptIds = await deptService.getChildrenDepts(this.userDeptId);
      return { deptId: { $in: childDeptIds } };
    case '5': // 仅本人数据权限
      return { createdBy: this.userId };
  }
};
```

### 2. 在服务层使用数据权限

```typescript
// 用户服务示例
async getUserList(query: any, dataScope?: any) {
  const conditions: any = {};

  // 关键词搜索
  if (query.keyword) {
    conditions.$or = [
      { account: new RegExp(query.keyword, 'i') },
      { username: new RegExp(query.keyword, 'i') }
    ];
  }

  // 数据权限过滤
  if (dataScope?.filter && Object.keys(dataScope.filter).length > 0) {
    Object.assign(conditions, dataScope.filter);
  }

  return await UserModel.find(conditions);
}
```

## 🐛 已知问题与解决方案

### 问题1：超级管理员权限检查

**现象**：超级管理员访问需要权限的接口返回403
**原因**：权限中间件只检查 `admin` 角色，未检查 `super_admin` 角色
**解决**：已修复权限中间件，同时检查 `admin` 和 `super_admin` 角色

### 问题2：部门模型中间件错误

**现象**：TypeScript编译报错
**原因**：中间件函数签名不匹配
**解决**：修复中间件函数签名，添加正确的类型定义

### 问题3：服务实例化问题

**现象**：部门控制器无法实例化服务
**原因**：服务类构造函数为私有，但控制器尝试直接实例化
**解决**：改为使用单例实例，通过默认导出使用
